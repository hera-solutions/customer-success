### Per-error detail

#### Error 1 — 403 storm on dashboard summary endpoints (UNCHANGED, GROWING)

Hera Lambda `dashboard` exposes 8 sub-routes under `/dashboard/associate/*-summary` (issue-trends, infraction, eoc, netradyne-alert, pps, events, license, opt-out) but they are not whitelisted in [`amplify/backend/api/heraPublicApi/cli-inputs.json`](amplify/backend/api/heraPublicApi/cli-inputs.json). API Gateway IAM rejects each call with `AccessDeniedException`. UI then floods with toasts and renders every dashboard card empty.

- **Hit count tripled since Run 1 (2026-06-05):** ~30 → ~143 hits, 7 → 46 users.
- **Action:** Open MSP ticket to add those 8 paths to `cli-inputs.json` and run `amplify push`. Same recommendation as Run 1; this is now the single biggest production pain.

#### Error 2 — Webpack ChunkLoadError (NEW; cascade)

LogRocket explicitly identifies this as a downstream effect of Error 1 — once auth fails mid-app, lazy-loaded chunks that need authenticated requests to resolve never finish. Fixing Error 1 should clear the bulk of these.

#### Error 3 — Cannot read 'tenant' on null (UNCHANGED, GROWING; cascade)

Same root cause family as Run 1's Error 6. The `userInfo.tenant` access throws when auth is partially torn down by the 403 storm. Stack frame names `useTenant.ts` / `AuthStore.ts` are LogRocket's source-map best-guesses; the actual hits are in tenant-dependent widgets across the Vue 2 hera app. Defense-in-depth guard still worth adding.

#### Error 4 — `/messenger/unread-messages` 504 (UNCHANGED, GROWING)

Backend Lambda response exceeds API Gateway 30s window for tenants with high message volume. Carry-over from Run 1's Error 5.

#### Error 5 — Daily Roster `notesByDate` TypeError (UNCHANGED — owner gap)

Top frame: [`src/views/DailyRoster/DailyRoster.vue`](src/views/DailyRoster/DailyRoster.vue) and [`multipleQueriesDailyRoster.js`](src/views/DailyRoster/multipleQueriesDailyRoster.js). This is the bug Abram filed as HERA-8534 on 2026-06-04 with priority None and no assignee. **No code commits in any branch since.** It has been live in prod for 4 days and is firing for 4 users.
